Getting Started with SecureHealth

This section will help you get started with the SecureHealth application, a HIPAA-compliant healthcare records system built with Symfony and MongoDB 8.2 Queryable Encryption.

What is SecureHealth?

SecureHealth is a comprehensive demonstration of how to build secure healthcare applications using modern encryption techniques while maintaining searchability and performance. It showcases MongoDB's Client-Side Field Level Encryption (CSFLE) and Queryable Encryption features.

Architecture Overview

SecureHealth uses a modern stack:

  • Backend: Symfony 7.0 framework with PHP 8.2
  • Database: MongoDB 8.2 with Queryable Encryption
  • Frontend: Simple React-based UI
  • Deployment: Docker containerization

Key Security Features

  • Field-level Encryption: Sensitive patient data is encrypted before it leaves the application
  • Searchable Encryption: Ability to query encrypted data without decrypting it first
  • Role-based Access: Different roles (doctor, nurse, receptionist) have different access levels
  • Audit Logging: All data access and modifications are logged for HIPAA compliance

System Requirements

  • Docker and Docker Compose
  • Git
  • 8GB RAM minimum (for Docker containers)
